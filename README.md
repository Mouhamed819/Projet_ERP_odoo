# Projet_ERP_odoo
Rapport de Projet ERP Odoo
1. Analyse et Conception
• Objectif : Centraliser et optimiser la gestion d'une entreprise dans un système unique.  
• Choix des Modules :
• Vente : Pour gérer tout le processus commercial (devis, commandes, factures) et le suivi des clients.  
• Signature (e-Sign) : Pour sécuriser et simplifier la signature des documents officiels avec une validité légale.  
• Rendez-vous (Meetings) : Pour planifier les interactions clients et automatiser les workflows.  
2. Réalisation (Sans Code)
• Configuration : Utilisation de l'interface dynamique (Frontend) basée sur JavaScript + HTML + CSS.  
• Intégration : Mise en place d'un flux où une action dans un module met à jour les autres automatiquement (ex: Signature liée à une vente).  
• Base de données : Toutes les informations sont stockées de manière sécurisée dans PostgreSQL.  
3. Stack Technique
• Backend : Python.  
• Base de données : PostgreSQL.  
• Frontend : JavaScript, HTML, CSS.  
• Serveur web : Nginx / Apache.  
4. Test et Déploiement
• Test : Réalisé sur GitHub via la documentation des flux métier.
• Déploiement : Solution accessible en ligne (Cloud), utilisable partout via une interface web.
